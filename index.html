This API provides flood prediction based on location-specific weather data. It uses machine learning models to process both historical and forecasted data.

Features of the API

It uses advanced data analysis and neural networks to provide very accurate flood risk predictions. By seamlessly integrating real-time weather forecasts with historical data for the past 23 days , the API thereby ensures that its predictions are on time and informed by current trends. This multi-layered strategy encompasses a comprehensive flood-risk analysis while taking into account rainfall, temperature variability, humidity, and wind patterns. Moreover, the API predicts up to 7 days in advance about weather conditions, which often makes the crucial difference between preparation and precipitation.

Trained on diverse and region-specific Indian meteorological datasets, this model presently achieves an accuracy rate of almost 97%. This high precision makes it a reliable source for solving the unique weather challenges that are prevalent in India, especially in flood-prone areas. The API is developer-friendly and can be used in different applications, such as weather monitoring platforms and disaster management systems.

Parameters Explained

    The API provides a detailed set of parameters for accurate flood prediction-

    Cloud Coverage: The average percentage of the sky covered by clouds. Higher values indicate overcast conditions.

    Humidity: The average atmospheric moisture level, expressed as a percentage.

    Maximum & Minimum Temperatures: The highest and lowest daily temperatures, measured in degrees Celsius.

    Wind Speed: The average wind speed in meters per second, which can influence weather patterns.

    Sunshine Duration: Sum total of hours in a day the place gets sunshine, influencing evaporation and conditions.

    Rainfall Sum : Accumulated millimeter sum of rainfall during the analyzed period.

    Flood Probability: Through this integration, the API returns all weather and flood-prone insights.

    All the data is retrieved from a public weather forecast API available at Open-Meteo .By combining these parameters, the API delivers comprehensive insights into weather patterns and flood risks.

Limitations

    Despite its accuracy, the Flood Prediction API has some limitations-

    Hosting Delay: Since the API is hosted on Render, the first call may take up to 50 seconds to fetch data due to server spin-up times.

    Surface Elevation: The API does not consider ground elevation. For example, areas below sea level are more prone to flooding but cannot be evaluated due to data limitations.

    Data Gaps: The training dataset lacks specific scenarios, such as ground-level flooding, making it challenging to predict certain cases.

    These are the limitations of the model and areas for improvement and potential integration with more comprehensive datasets in the future.

How to Use the API

    Follow these steps to integrate the API into your application:

    Copy the API endpoint using the "Copy URL" button. Make an HTTP GET request with latitude and longitude parameters in the query string. Parse the JSON response, which includes all relevant weather data and flood probabilities.

    Example response from the API- 

{
    "avg_cloud_coverage": 65,
    "avg_humidity": 80,
    "avg_max_temp": 34,
    "avg_min_temp": 26,
    "avg_wind_speed": 5.6,
    "latitude": 20.5,
    "longitude": 78.96,
    "sunshine_duration": 8,
    "total_rain_sum": 12.5,
    "flood_probability": 0.45
}

The flood_probability parameter ranges from 0 (no flood risk) to 100 (extremely high flood risk). Use this value to make informed decisions.
